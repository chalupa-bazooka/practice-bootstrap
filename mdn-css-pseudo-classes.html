Pseudo classes in CSS

As explained on MDN

In this page I'll focus on several types of pseudo classes: tree structure, user action, and functional.

Tree structure

:empty
  • Not empty

The idea here is that we apply the pink outline style to all empty <li> elements. The first element is obviously not empty. The second contains a whitespace and it's also considered not empty. The third, however, has nothing and thus is viewed as empty, hence it gets the specified style.

:nth-child()
  • First child
  • Second child
  • Third child
  • Fourth child
  • Fifth child

The even children have colored bgs. The second child has outline. And each first child starting with the third one is bold.

:nth-last-child()
  • First child viewed as last
  • 2nd child viewed as 4th
  • 3rd child viewed as 3rd
  • 4th child viewed as 2nd
  • Last child viewed as first

nth-last-child starts counting from the end. Hence the second element has the outline. The last three are bold.

Both lists belong to the same class. But only one of them has background for its <li> – that's because this style is targeted at only-child.

The background-color style is set to apply only to odd <div> children elements. Hence a <p> element is disregarded.

User action

:hover

The background color of the button changes to green upon hover.

:active

When you press this button, its background-color turns purple for a moment. You can press the button without releasing it to see the color better.

:focus

When you take this input in focus (click its field), its background color changes.
